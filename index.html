Dapplets

Dapplets is the protocol for permissionless, decentralized and privacy-aware Augmented Web as well as its reference implementation.

Specification

Dapplets is a crypto-economic standard and defines both technical interfaces for injecting Dapps into hosting websites as well as user roles and incentive structure for actors in the ecosystem. The specification is intentionally made as lean and generic as possible. Any aspects, that are not necessary to bring parts working together are left up to implementers or have a recommendation status.

Overview

Dapplets protocol enables creation of unstoppable augmentations for third party content. It could be:

  1. Ratings, feedbacks or other identity based services provided by community.
  2. Scam and Fake News warnings provided by community.
  3. Crypto-economic addons for existing website communities.
Use SOWA to provide secured visual experience while signing these crypto actions. Switch on the dapplet on this site to get an idea what is about.

Implementation

Dapplets is currently implemented as a browser plugin but we are looking for cooperation to implement it in a browser natively. For details of the existing implementation please see our GitHub. If you are developer working on the (dapp) browser, let us know.

Community

There are different roles in the Dapplets Project:

  1. CoreDevs are working on the protocol, its reference implementation and incentive structure.
  2. ModuleDevs are implementing modules and adapters used by Applications.
  3. AppDevs are implementing own business models on the top of Dapplets protocol.
  4. Auditors are verifying modules created by developers and signal to the users an audit status.
  5. Users are free to follow any auditor and use any apps existing in the ecosystem.
