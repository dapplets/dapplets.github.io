Dapplets

Dapplets is a protocol for permissionless, decentralized and privacy-aware Augmented Web.

Specification

Dapplets is a crypto-economic standard defining data interfaces and community roles for injecting Dapps into websites.

The specification is intentionally made as lean and generic as possible. Any aspects, that are not necessary to bring parts working together are left up to implementers or have a recommendation status.

Overview

Dapplets protocol enables creation of unstoppable augmentations for third party websites. It could be:

  1. Identity services: security alerts, ratings and feedbacks.
  2. Scam and fake news alerts.
  3. Payments and other crypto-economic addons for existing web communities.
Do you have more ideas for Dapplet Apps? Join our community and create your own business case!

Implementation

Dapplets is currently implemented as a browser plugin and as a javascript library (like on this site), but we are looking for cooperation to implement it in a browser natively.

For details of the existing implementation please see our GitHub. If you are developer working on the (dapp) browser, let us know.

Community

There are different roles in the Dapplets Project:

  1. CoreDevs are working on the protocol, its reference implementation and incentive structure.
  2. ModuleDevs are implementing modules and adapters used by Applications.
  3. AppDevs are implementing own business models on the top of Dapplets protocol.
  4. Auditors are verifying modules created by developers and signal to the users an audit status.
  5. Users are free to follow any auditor and use any apps existing in the ecosystem.
