Secured Open Wallet Architecture (SOWA)

Make the WYSiWYS principle great again!

is a new standard for Crypto Wallets ensuring WYSiWYS security for their Users and enabling more powerful yet intuitive workflows for Dapp Developers.

DISCLAIMER: Work in progress!

Specification

A reference implementation acts currently as a the specification de-facto. A dedicated specification is in progress.

Overview

SOWA solves the fundamental trade-off problem between UX and Security as described here (medium article) and creates many advantages for both Wallet and Dapp developers as well as for Users. SOWA and Dapplets are symbiotic project. SOWA allows more powerful and secure workflows for Dapplets, while Dapplels create incentives for Wallets to implement SOWA.

Implementation

A reference implementation of the SOWA protocol made in typescript is the part of the Dapplets Project. This integrated Wallet pops up if the target wallet has no SOWA-capabilities and then acts as a proxy. If you are Wallet Dev, you are free to integrate or re-implement the reference SOWA-Wallet into your wallet.

for Wallet Devs

Why you should support SOWA

  • access to user base on legacy sites provided by Dapplets
  • better UX and Security
  • community support for non-standard Tx
  • support for multisigs and other Tx-postprocessing
  • better support for wallet permissions

Try it now!

The fastest way to try out the SOWA in action is to install SOWA-aware wallet and switch dapplets on on this site. For full experience install Dapplets extension (or take the Dapplets-aware browser) and start browsing.

for Dapp Devs

Unique advantages to adopt SOWA:

  • better UX without compromise on security
  • WYSiWYS transaction signing
  • dapps from static sources (QRCode)
  • heterogenous transaction batching
